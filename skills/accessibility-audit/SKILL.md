# Accessibility Audit Skill

Use Browser DevTools MCP accessibility (a11y) tools to audit pages for accessibility.

## When to Use

- User asks about accessibility or a11y
- User mentions WCAG compliance or screen reader compatibility
- User needs to audit ARIA roles and labels
- User asks about keyboard navigation or focus order

## Guidance

1. Use ARIA snapshot tools to get a compact, ref-based view of the page for interactions.
2. Use AX tree snapshot tools for detailed accessibility tree analysis (roles, names, states).
3. Optionally filter by roles, check occlusion, or only visible elements as needed.
4. Use interaction tools (e.g. press key) to test keyboard navigation when relevant.
