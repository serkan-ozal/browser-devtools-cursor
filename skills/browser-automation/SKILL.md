# Browser Automation Skill

Use the **Browser DevTools MCP** tools for browser automation, testing, and debugging.

## When to Use

- User needs to navigate to a URL, take a screenshot, or extract page content
- User wants to automate clicks, form filling, or keyboard input
- User asks about accessibility (a11y), ARIA, or AX tree
- User needs Web Vitals, console logs, or HTTP request inspection
- User wants to mock API responses or intercept requests
- User needs to run JavaScript in the browser or inspect React components
- User wants to compare the page with a Figma design or do visual testing

## Capabilities (via MCP tools)

The plugin provides MCP tools for:

- **Navigation** – go to URL, back, forward, reload
- **Content** – screenshot, PDF, get HTML/text
- **Interaction** – click, fill, hover, scroll, drag, press key
- **Accessibility** – ARIA snapshot, AX tree snapshot
- **Observability** – Web Vitals, console messages, HTTP requests, trace ID
- **Stub** – intercept and mock HTTP requests
- **Run** – execute JavaScript in browser or sandbox
- **React** – get component for element, element for component
- **Figma** – compare page with design
- **Debug** – tracepoints, logpoints, exception monitoring

Use the available MCP tool names (e.g. navigation, content, interaction, a11y, o11y, stub, run, react, figma, debug) as provided by the Browser DevTools MCP server.
