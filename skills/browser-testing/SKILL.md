# Browser Testing Skill

Use Browser DevTools MCP tools for automated browser testing and interaction automation.

## When to Use

- User asks to test a web page or application
- User wants to automate browser interactions (click, fill, select)
- User needs to verify UI behavior or form validation
- User mentions multi-step forms, wizards, or login/signup flows

## Guidance

1. Navigate to the target URL first.
2. Use accessibility (ARIA/AX) snapshot tools to get stable element refs when possible.
3. Use interaction tools for click, fill, select, hover, scroll, drag, and keyboard.
4. Use content tools for screenshots or HTML/text when verifying output.
5. Use sync/wait tools when waiting for network or content to stabilize.
